# HLV_TrafficLights — Техническое задание

## 1. Цель проекта

Игрушечная система светофоров для детской дороги-пазла. Одновременно — учебный проект для изучения STM32 в Arduino-среде и потенциальный контент для канала Home Lab Vibes.

---

## 2. Функциональные требования (v1.0)

### 2.1 Светофоры
- **4 независимых светофора**, каждый со своим модулем LED (R, Y, G) и **своей кнопкой**
- Каждый светофор работает **полностью автономно** — своя кнопка, свои тайминги
- Фазы цикла: 🔴 Красный → 🔴+🟡 (подготовка) → 🟢 Зелёный → 🟡 Жёлтый → 🔴
- Тайминги красного и зелёного — **индивидуальные** для каждого светофора (настраиваемые константы)
- Тайминг жёлтого — общий для всех

### 2.2 Поведение при включении
- Все светофоры **сразу стартуют в рабочем режиме**, начиная с 🔴 красного
- Никакого промежуточного состояния — ребёнок включил, всё работает

### 2.3 Управление (по одной кнопке на светофор)

```
Включение → [Рабочий режим 🔴→🟡→🟢] 
                ↕ кнопка
         [Аварийка ⚠️ мигает 🟡]
```

- **Нажатие кнопки** в рабочем режиме → переход в аварийный (мигающий жёлтый)
- **Повторное нажатие** → возврат в рабочий режим (начиная с 🔴 красного)
- Каждый светофор переключается **независимо** от остальных

### 2.4 Будущие фичи (v2.0+)
- Дополнительные алгоритмы переключения (циклы через аварийку)
- Пешеходная кнопка, зуммер
- Привязка к конкретной конфигурации дороги-пазла

## 3. Аппаратная часть

### 3.1 Микроконтроллер
- **STM32F411CEU6 (Black Pill)** — ARM Cortex-M4, 100 МГц, USB-C
- Среда разработки: **Arduino IDE** + ядро STM32duino

### 3.2 Модули светофоров
- **Модуль:** LED Traffic Light Module (V1224), набор из 5 шт. (используются 4, 1 в запасе)
- **Размеры:** 56 × 21 × 11 мм
- **LED:** 5 мм × 3 (красный, жёлтый, зелёный)
- **Интерфейс:** common cathode, 4 пина: GND, R, Y, G
- **Напряжение:** 5V (номинал), резисторы на плате
- **Совместимость с 3.3V GPIO:** LED загорятся, но тусклее (~60–70% яркости). Для игрушки приемлемо. При необходимости полной яркости — транзисторные ключи (NPN 2N2222 или ULN2803/ULN2004)
- **Крепление:** 2 монтажных отверстия на каждом модуле

### 3.3 Кнопки
- **4 тактовые кнопки** (по одной на каждый светофор) с программным debounce
- Подключение: пин → кнопка → GND (используем внутренний INPUT_PULLUP)

### 3.4 Питание

- Планируется: **аккумулятор 18650** (3.7V ном. / 4.2V макс.)
- Black Pill ожидает 5V на пин 5V; модули светофоров также рассчитаны на 5V
- **Рекомендуемая схема:** Boost-конвертер (MT3608) 3.7V → 5V
  - 5V → пин 5V Black Pill (через встроенный регулятор → 3.3V для MCU)
  - GND общий для всей системы
  - GPIO 3.3V на сигнальные пины модулей (LED будут работать, чуть тусклее)
  - При необходимости полной яркости: 5V через транзисторные ключи, управляемые GPIO 3.3V

> **Потребление (оценка):** 4 модулей × 3 LED × ~15 мА = ~180 мА макс. (все LED горят) + Black Pill ~50 мА ≈ **~230 мА**. Одной 18650 (2500–3000 мАч) хватит на **~11–13 часов** непрерывной работы.

### 3.5 Конструктив и подключение
- Используются **готовые модули V1224** — отдельные платы не нужны
- Подключение к Black Pill через **4-пиновые кабели** длиной до 1 метра (GND, R, Y, G)
- В наличии **4 комплекта метровых проводов**
- Кнопки подключаются 2-пиновым кабелем (пин + GND)
- Разъёмы: *(TBD — JST-XH / DuPont / другие)* — светофоры съёмные
- Модули имеют монтажные отверстия — можно крепить к подставкам/столбикам

### 3.6 Распиновка (финальная)

**Правая сторона платы:**

| Светофор | R (красный) | Y (жёлтый) | G (зелёный) | Кнопка |
|----------|-------------|------------|-------------|--------|
| TL1      | PA0         | PA1        | PA2         | PA3    |
| TL2      | PA4         | PA5        | PA6         | PA7    |

**Левая сторона платы:**

| Светофор | R (красный) | Y (жёлтый) | G (зелёный) | Кнопка |
|----------|-------------|------------|-------------|--------|
| TL3      | PB12        | PB13       | PB14        | PB15   |
| TL4      | PB3         | PB4        | PB5         | PB6    |

**GND:** пин G на соответствующей стороне платы (общий для модулей и кнопок)

**Зарезервировано:**
- PA11/PA12 — USB D-/D+ (прошивка/питание)
- PA13/PA14 — SWD (отладка)
- PB8/PB9 — I2C1 SCL/SDA (будущий OLED дисплей)
- PC13 — встроенный LED (индикация статуса)

**Свободно для будущих фич:** A8, A9, A10, A15, B0, B1, B2, B7, B9, B10, C14, C15 (~12 GPIO)

## 4. Программная архитектура

### 4.1 Среда и зависимости
- **Arduino IDE** + **STM32duino** (пакет плат STM32)
- Board: Generic STM32F4 → BlackPill F411CE
- Внешние библиотеки: не требуются (v1.0)

### 4.2 Структура кода (план)

```
HLV_TrafficLights/
├── HLV_TrafficLights.ino    // main: setup(), loop()
├── config.h                  // пины, тайминги для каждого светофора
└── traffic_light.h/.cpp      // класс/структура одного светофора (LED + кнопка + логика)
```

### 4.3 Логика работы
- Неблокирующая архитектура на `millis()` (без `delay()`)
- Каждый светофор — структура/класс, содержащий:
  - Пины: R, Y, G, кнопка
  - Индивидуальные тайминги: длительность красного и зелёного
  - Текущее состояние: фаза цикла + режим (рабочий/аварийный)
  - Время последнего переключения фазы
- Кнопка с debounce (~50 мс), переключает режим своего светофора
- При включении: все светофоры стартуют в рабочем режиме с 🔴 красного
- Возврат из аварийки в рабочий режим — всегда начинается с 🔴 красного
- Масштабирование: добавить светофор = добавить запись в массив конфигурации
